  ## 关键技术选型（摘要）

  - 前端：React 或 Vue，文件上传使用 presigned URL，状态更新可用 WebSocket 或轮询。
  - 后端 API：Spring Boot（Spring Web），鉴权使用 Spring Security + JWT，参数校验使用 Bean Validation。
  - 业务层：Spring Services，数据库访问使用 Spring Data JPA 或 MyBatis，迁移工具 Flyway / Liquibase。
  - 异步队列：RabbitMQ（Spring AMQP），Python 端使用 Celery（或 FastAPI + gRPC）作为 worker。
  - 存储：MySQL 存储元数据与任务状态；MinIO / AWS S3 存储大文件（原图、处理结果）。
  - 观测与运维：micrometer -> Prometheus, Grafana；错误追踪 Sentry；日志 ELK / Loki；容器化 Docker + Kubernetes，CI 使用 GitHub Actions。

  ## 层级说明与开发要点

  - API 层（Controller）: 保持瘦控制器，做输入校验、鉴权和错误映射；暴露 `POST /api/projects/{id}/run`、`POST /api/tasks/{id}/callback` 等接口。
  - 业务层（Service）: 管理事务边界（`@Transactional`），实现任务创建、分发与状态管理；采用 outbox 模式或先写 DB 后发消息保证可靠性。
  - 图像处理引擎（Worker）: 将现有 Python 脚本封装为 Celery 任务或独立服务，确保幂等、重试与分块处理大图。
  - 存储层: 使用 presigned URL 让客户端/worker 直接上传/下载对象存储，后端仅保存 `storage_key` 引用；对 `params`/`payload` 使用 JSON 列。
