# # **区域识别模块设计文档**

（文档头部补充说明）
> 注：Python 算法模块统一放置于仓库的 [color-analysis-python/](color-analysis-python/) 目录下，后端 Worker 从该目录加载算法实现。

## **2. 核心职责与流程**

### **2.1 核心职责**
1.  **图像预处理：** 对输入图像进行灰度化、降噪处理。
2.  **边缘检测：** 根据配置调用不同的算法识别图像轮廓。
3.  **区域提取：** 将闭合轮廓转换为多边形（polygon）表示的区域集（顶点按顺序排列，归一化坐标）。
4.  **点选识别（前端本地）：** 前端使用后端返回的多边形集合在客户端进行点内测试，以减少前后端交互。
5.  **掩码生成：** 为后端分析生成二值化掩码（Mask）。

## **3. 接口定义与数据流**

### **3.1 Spring Boot API (自动区域识别任务提交)**

*   **Endpoint:** `POST /api/tasks/region-recognition`
*   **请求体 (Request Body):**
    ```json
    {
      "image_id": "uuid-string",
      "storage_key": "path/to/image.png",
      "algorithm_config": {
        "method": "CANNY",
        "threshold1": 100,
        "threshold2": 200,
        "gaussian_blur": 5,
        "min_area": 500
      }
    }
    ```
*   **说明：** 不再传入光标坐标，后端只接收图像标识/存储 key 与算法配置。

### **3.2 Python Celery Worker (自动区域识别任务处理)**

*   **输入 (Input):** 接收来自 RabbitMQ 的任务消息，包含 `image_id`, `storage_key`, `algorithm_config`。
*   **输出 (Output):** 处理完成后，通过 HTTP POST 请求回调 Spring Boot 的 `/api/tasks/{task_id}/callback`，提交识别到的区域集合（多边形列表，归一化坐标）。

### **3.3 Spring Boot API (结果回调 - 自动识别)**

*   **Endpoint:** `POST /api/tasks/{task_id}/callback`
*   **请求体 (Request Body)：**
    ```json
    {
      "status": "success",
      "result_payload": {
        "regions": [
          {
            "region_id": "r1",
            "type": "polygon",
            "points": [
              {"x": 0.12, "y": 0.34},
              {"x": 0.15, "y": 0.38},
              {"x": 0.18, "y": 0.33}
            ],
            "bounding_box": {"x": 0.1, "y": 0.3, "w": 0.05, "h": 0.08}
          }
        ],
        "metadata": {
          "original_width": 1920,
          "original_height": 1080
        }
      },
      "logs": "Optional logs or error messages"
    }
    ```
*   **说明：** 每个 region 的 `points` 表示多边形顶点顺序（建议顺时针或逆时针一致），所有坐标均归一化至 [0.0, 1.0]。

### **3.4 Spring Boot API (前端获取区域多边形)**

*   **Endpoint:** `GET /api/images/{image_id}/regions`
*   **响应体 (Response Body):**
    ```json
    {
      "regions": [
        {
          "region_id": "r1",
          "type": "polygon",
          "points": [ {"x":0.12,"y":0.34}, ... ],
          "bounding_box": {"x":0.1,"y":0.3,"w":0.05,"h":0.08}
        }
      ],
      "metadata": { "original_width":1920, "original_height":1080 }
    }
    ```
*   **说明：** 前端获取多边形集合后在客户端完成点选（point-in-polygon）判断与高亮，避免频繁后端交互。若需要服务器端点选匹配，可视为可选扩展 API。

## **4. 前后端职责与交互**

### **4.1 后端 (陈祚垟)**
*   **输入：** 图像文件（通过 `storage_key` 从 MinIO 获取）。
*   **输出：** 区域多边形集合（JSON，归一化）。
*   **任务：**
    1.  实现 Python Celery Worker（放置于 [color-analysis-python/](color-analysis-python/)），负责消费 RabbitMQ 任务，执行自动区域识别并输出多边形集合。
    2.  封装算法插件，实现图像预处理、边缘检测、区域提取（输出多边形）。
    3.  将多边形序列化并回调 Spring Boot，存储可通过 `GET /api/images/{image_id}/regions` 访问的多边形集合。
    4.  确保多边形顶点顺序一致（CW 或 CCW），并对多边形进行简化以减少传输量。
    5.  支持任务重试、超时与日志记录。

### **4.2 前端 (王语欣)**
*   **输入：** 用户在 Canvas 上的点击坐标（Canvas 坐标）。
*   **输出：** 在客户端完成点选匹配并高亮多边形区域。
*   **任务：**
    1.  获取 `GET /api/images/{image_id}/regions` 返回的多边形集合并在 Canvas 上渲染（按归一化坐标映射）。
    2.  实现客户端的 point-in-polygon 判断以响应用户点击并高亮区域（减少一次网络往返）。
    3.  支持多边形的本地微调（拖拽、顶点编辑）并将修改结果通过 API 保存回后端（若需要）。
    4.  处理多边形渲染的缩放与像素对齐问题。

## **5. 算法实现细节**

### **5.1 算法解耦设计**
采用策略模式（Strategy Pattern）封装算法，算法实现放在 [color-analysis-python/](color-analysis-python/)：
*   **Canny 策略**
*   **Sobel 策略**
*   **自定义策略**（预留 SAM 等）

### **5.2 跟进要点 (兼容性保障)**
1.  **坐标系对齐：** 原点 (0,0) 统一为图像左上角。
2.  **数据格式：** `points` 数组应按顺序给出顶点（CW 或 CCW），便于前端直接做 fill/渲染。
3.  **多边形简化：** 对高顶点数多边形进行 Douglas–Peucker 简化（或类似方法），以减少网络负载。
4.  **性能优化：** Worker 对高分辨率图片进行等比例缩放识别，识别后映射回原图坐标并返回归一化值。
5.  **点选识别：** 推荐把点选逻辑放在前端（client-side point-in-polygon），后端保留可选的服务器端点选接口作为兼容。

## **6. 验收标准（更新）**

1.  **准确性 (自动识别)：** 标准模板自动识别的区域覆盖率应 ≥ 95%。
2.  **点选交互：** 前端在任意已识别区域内部点击，应能在客户端准确高亮对应多边形（客户端点选延迟 < 100ms）。
3.  **一致性：** 同一张图片多次识别结果坐标集应稳定一致。
4.  **数据体积：** 返回的多边形数据在常见场景下应通过简化后减少 ≥ 50% 的顶点数（视数据而定）。
5.  **性能 (自动识别)：** 单张 1920x1080 图片自动识别（含缩放）从提交到回调耗时目标 ≤ 5s。
6.  **接口响应：** `GET /api/images/{image_id}/regions` 的响应时间目标 ≤ 200ms（缓存/索引支持）。
